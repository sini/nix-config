# Flake-File Migration Design

**Date:** 2026-04-02 **Status:** Approved

## Overview

Migrate from manual flake.nix to vic/flake-file pattern for flake generation.
This involves moving all flake inputs (71 total), description, and nixConfig settings
from the top-level flake.nix into the flake-parts module system using
flake-file.nix.

## Motivation

The flake-file pattern provides:

- Modular flake configuration through the Nix module system
- Type-safe input and output schemas
- Automatic flake.nix generation
- Better organization for large flake configurations
- Future capability to split inputs across multiple modules

## Scope

### In Scope

- Migrate all 71 inputs from flake.nix to flake-file.inputs
- Migrate description to flake-file.description
- Migrate nixConfig to flake-file.nixConfig
- Sort all inputs alphabetically
- Add inline comments to maintained forks (5 sini forks only)
- Preserve all input.\*.follows relationships exactly
- Remove/comment out `den` input and import from flake-file.nix

### Out of Scope

- Modifying the generated flake.nix (flake-file will regenerate it)
- Splitting inputs into separate modules (future work)
- Using the `den` input and its flakeModule (deferred for later - will be removed/commented out)
- Modifying any module configurations in ./modules

## Architecture

### File Structure

**Modified:**

- `modules/flake-parts/devtools/flake-file.nix` - Expand from 3 example inputs
  to complete configuration

**Not Modified:**

- `flake.nix` - Will be regenerated by flake-file automatically

## Implementation Plan

### Step 1: Extract Current Configuration

Read flake.nix to extract:

- Description (top-level `description` attribute)
- nixConfig (top-level `nixConfig` attribute)
- All inputs (top-level `inputs` attribute set, approximately 70 inputs)

### Step 2: Transform to flake-file Format

- Sort all inputs alphabetically
- Transform each input to flake-file.inputs.<name> syntax
- Add inline comments to the 5 maintained forks
- Preserve exact URL strings and follows relationships

### Step 3: Write flake-file.nix

Replace the current example content with complete configuration including:

- imports section with only flake-file.flakeModules.dendritic (comment out the den.flakeModules.dendritic import)
- flake-file.description (preserve exact wording)
- flake-file.nixConfig (preserve settings, clean up comment formatting)
- flake-file.inputs (all 71 inputs from flake.nix, remove the example den input from flake-file.nix)

### Step 4: Verification

1. Run flake-file generation command (likely `nix run .#write-flake`)
1. Compare generated flake.nix against original to ensure all inputs present
1. Run `nix flake check` to verify flake validity
1. Test with `nix-flake-build <hostname>` to ensure builds work

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Typo in input name during sorting | Build failure | Systematic copy-paste, verify all ~70 inputs present |
| Incorrect follows relationship | Dependency issues | Preserve exact structure from original |
| Missing input | Build failure | Compare generated vs original flake.nix |
| flake-file tool issues | Cannot generate flake.nix | Test generation command early in process |

## Success Criteria

- Exactly 71 inputs migrated to flake-file.nix (all inputs from current flake.nix)
- Description and nixConfig migrated with exact settings preserved
- Alphabetical ordering maintained across all inputs
- Inline comments added to 5 maintained sini forks
- All inputs.\*.follows relationships preserved exactly
- `den` input removed from flake-file.nix (was in example, not in flake.nix)
- `den.flakeModules.dendritic` import commented out in flake-file.nix
- Generated flake.nix passes `nix flake check`
- Test build succeeds (e.g., `nix-flake-build cortex`)
